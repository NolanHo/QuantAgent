## 关联

- Issue: #169
- OpenSpec change: `add-source-jina-reader`

## 背景

`#169` 的目标不是补 `Readability` 实现，也不是接 Runtime、ToolRegistry、SourceBinding 或 Event Bus，而是先把官方 `Jina Reader` Source Plugin 的 OpenSpec 边界和最小交付物收清，作为与 `Readability Link Reader` 平行的第二条 reader 路径。

当前仓库已经通过 `#137`、PR `#145`、`docs/design/06-source-plugin-design.md` 和 `docs/design/11-crawler-source-plugin-boundary.md` 收住了 crawler / reader 类 Source Plugin 的职责边界；`#139 / #155 / #164` 则提供了 `Readability` 这条并行 reader 先例。本 PR 在这些既有边界上新增 `Jina Reader` 的 OpenSpec change，不重写全局 Source Plugin 契约。

## 改动

### OpenSpec change artifacts

- 新增 `openspec/changes/add-source-jina-reader/`
- 补齐 `proposal.md`、`design.md`、`tasks.md`、`.openspec.yaml`
- 新增 `specs/source-jina-reader/spec.md`

### 本轮固定的关键决策

- `Jina Reader` 本轮只做官方 `source/read` 插件，不同时暴露 `tool.read_url`
- 插件输出直接贴齐现有 source runtime 可消费的 `RawEventDraft` 兼容 DTO
- 插件公开 schema 只声明最小非敏感配置字段；外部 reader 鉴权由平台统一控制
- 私有链接、受限内容或默认不应外发的 URL，本轮默认禁止发给外部 reader
- 外部 reader 限流、超时、不可用时，本轮只要求清晰失败返回，不引入自动 fallback 到 `Readability`

### 和既有 reader 链路的关系

- 本 change 把 `#139 / #155 / #164` 作为并行 reader 先例，而不是硬阻塞前置
- 本 PR 不修改 `Readability` 插件的范围、实现或验收口径
- 本 PR 不实现 `Jina Reader` 代码，只为后续实现 PR 提供经审核的 OpenSpec 边界

## 风险与未验证项

- 本 PR 只定义 `Jina Reader` Source Plugin 的契约和设计边界，没有实现插件代码。
- 平台后续以什么 secret / policy 入口向 `Jina Reader` 提供外部服务鉴权，本 change 只锁定“平台统一控制”，未细化到实现字段。
- 敏感 URL 的具体判定规则后续可能需要独立 policy / allowlist change 承接。
