# Change: App Error Boundary

## Problem

当前 Web 应用需要在根部统一处理启动期异常、子树渲染异常和运行时未捕获异常，避免白屏，并提供最小恢复入口。实现必须保持最小范围，只覆盖本 issue 的兜底能力，不扩展到 API、Query 或业务级错误治理。

## Approach

复用现有 `bootstrap.tsx`、`error-boundary.tsx` 和路由壳集成点，补齐一个统一错误页与启动失败兜底页。错误展示只保留摘要和可选标识信息，恢复动作只提供重新加载和返回默认入口。实现时优先保持现有启动流程不变，只在必要位置插入错误兜底。

## Tasks

- 梳理现有 Web 启动链路和根路由挂载点。
- 设计统一错误展示数据结构，限定可见字段。
- 实现或调整根级 Error Boundary，覆盖渲染异常。
- 实现启动失败兜底页，覆盖 bootstrap 阶段异常。
- 接入最小恢复动作：reload 与返回默认入口。
- 确保错误页不会泄露堆栈、密钥或环境配置。
- 进行最小验证，确认启动失败与子树异常都能落到统一页面。

## Notes

- 不处理 API Client、Query、WebSocket、路由页面业务错误分层。
- 不引入复杂错误分类、上报或埋点机制。
- 默认入口以当前应用首页或根入口为准。
