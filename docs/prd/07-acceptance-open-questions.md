# 07. 验收标准与待确认项

## 信息完整度评估

| 维度 | 状态 | 说明 |
| --- | --- | --- |
| 目标或预期业务结果 | 部分明确 | 明确要做事件驱动量化 Agent，但缺少可量化业务指标 |
| 目标用户或操作角色 | 明确 | 面向专业交易者、研究员和策略开发者 |
| 范围内行为 | 明确 | 包含事件监控、路由、行业推理、辩论、HITL 和通知/执行 |
| 范围外边界或限制 | 部分明确 | 已明确部分市场无 API 权限，但交易市场、账户范围和合规边界未定义 |
| 成功标准或验收条件 | 部分明确 | 已整理跨模块验收标准，但仍缺少量化指标 |
| 时间、发布、合规约束 | 缺失 | 未定义发布时间、上线门禁、监管和真实交易合规要求 |

## 跨模块验收标准

### 产品验收

- 用户能够配置至少一个行业包并看到其监控目标。
- 用户能够看到事件从捕获、路由、推理、验证到建议的完整状态。
- 用户能够理解每条交易建议的证据、反方观点、风险和置信度。
- 用户能够确认、拒绝或要求重新分析 HITL 建议。

### 行业包验收

- `Oil_Industry_Package` 能处理典型石油制裁事件，并输出一级影响、二阶影响和候选交易逻辑。
- `Memory_Package` 能处理典型 AI 显存优化论文事件，并输出一级影响、二阶影响和候选交易逻辑。
- 每个行业包都需要包含 Watcher、Logical Graph 和 Strategy Adapter。
- 每个行业包都需要暴露 MCP 工具和资源，并提供健康状态。

### Agent 流程验收

- Router Agent 能从非结构化文本中提取关键实体。
- Router Agent 能给出行业包路由理由。
- Internal Debate 至少输出支持观点、反对观点和不确定性。
- 系统能根据置信度进入自动执行、人工确认或仅通知模式。

### 风控验收

- 高风险二阶推理必须完成至少两个独立信源验证。
- 验证失败时不得自动执行。
- 单一来源高频噪音必须触发熔断或降级。
- 无 API 权限市场不得出现直接自动下单。
- 敏感配置不得出现在前端、日志或仓库文件中。

### UX 验收

- 思维链视图能展示关键推理步骤和状态。
- 行业包管理看板能展示监控目标、关键词、启停状态和最近触发事件。
- 交易建议弹窗能展示建议动作、证据、反方观点、风险和用户操作。
- 错误态需要说明失败阶段并提供可重试或反馈路径。

## 测试重点

- 典型石油事件 happy path。
- 典型半导体/内存事件 happy path。
- 一个事件触发多个行业包的跨行业场景。
- 单一账号高频发文触发噪音熔断。
- 数据源验证失败导致降级为人工确认或仅通知。
- 置信度阈值边界：低于 70%、70%-90%、高于 90%。
- 无交易 API 权限时的仅通知模式。
- 用户拒绝建议后系统不得继续自动执行。
- WebSocket 断线重连后的状态恢复。

## 顶层风险

| 风险 | 影响 | 建议 |
| --- | --- | --- |
| 自动交易边界不清 | 可能导致不符合权限或合规要求的执行 | 在实现前定义市场、账户、权限、风控开关和人工审批规则 |
| 二阶推理难验证 | 可能产生高置信度但错误的交易建议 | 强制多信源验证，并保留反方观点和不确定性 |
| 行业包知识图谱过粗 | 容易把复杂行业关系简化为错误因果链 | 为每个行业包维护版本化逻辑图谱和样例集 |
| CoT 展示边界不清 | 可能暴露过多模型内部推理或形成不可控解释 | UI 展示可解释推理摘要，不默认展示完整原始推理 |
| 数据源稳定性不足 | 可能影响事件及时性和可信度 | 对数据源建立健康检查、失败重试和来源置信度 |

## 待确认问题

1. v1 首批真实接入的交易市场和金融工具范围是什么，是否包含真实自动下单？
2. 自动执行是否需要额外的账户级风控，例如最大仓位、最大亏损、冷却时间和每日次数限制？
3. 两个独立信源的定义是什么，官方源、新闻源、社交媒体、历史数据库是否有优先级？
4. 行业包逻辑图谱由谁维护，是否需要审核流程和版本发布机制？
5. UI 中展示的推理链路应到什么粒度，是否需要隐藏模型原始推理，仅展示用户可审计摘要？
6. Discord、Telegram 通知是否是 v1 必需能力，还是后续集成？
7. 历史相似事件回测需要覆盖多久的数据，评价指标是什么？

## 是否建议进入实施

建议进入原型实施和技术验证，但不建议直接进入真实自动交易上线。

理由：

- 产品方向、核心模块和关键流程已经明确，足够支持 Router Agent、行业包、事件流和 HITL 原型。
- 自动执行、交易权限、资金风控、合规边界和多信源定义仍不完整，不足以支撑真实交易闭环上线。
- 首阶段应优先实现仅通知和人工确认模式，自动执行能力应在风控规则明确后单独评审。
