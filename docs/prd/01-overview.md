# 01. 产品概览

## 基本信息

**产品名称**：QuantAgent  
**版本**：v1.2.0  
**状态**：正式整理稿  
**核心架构**：FastAPI + DeepAgents + React + PostgreSQL + 插件运行时

## 项目定义

QuantAgent 是一款面向专业交易者和研究员的事件驱动型量化智能系统。系统通过 Source Plugin 采集外部信息，经 Router Agent 路由到行业包，再由 AgentRuntime、Scoring / Debate 和 Decision / Policy Gate 输出可审计的建议、通知或受控执行请求。

产品核心能力是围绕事件链进行二阶分析：从事件本身出发，推导一级影响、二阶影响和候选交易逻辑，并将结果转化为可确认或可执行的动作建议。

## 核心愿景

系统采用插件化设计。Source、Industry、Strategy、Notification、Executor 都按插件体系规划，官方插件、第三方插件、私有插件和运行时安装插件通过 Registry 分层管理。

目标不是构建通用聊天机器人，而是构建一个围绕事件监控、路由、行业分析、评分、审批和受控执行的量化 Agent 系统。

## 目标用户

- 专业交易者：需要快速得到可解释的事件分析和动作建议。
- 量化研究员：需要沉淀行业因果链并验证事件信号。
- 策略开发者：需要以插件方式扩展行业包、数据源、工具和执行能力。

## 范围内能力

- 监控外部事件源，包括社交媒体、官网、新闻源、论文平台和行业公开信息。
- 从非结构化文本中提取实体、事件、行业和潜在影响对象。
- 通过 Router Agent 将事件路由到一个或多个行业包。
- 使用行业包和 AgentRuntime 形成结构化行业分析。
- 通过 Scoring / Debate 输出置信度、支持观点和反方观点。
- 依据权限、风险和用户策略进入通知、人工确认、限时确认、dry-run 或受控执行。
- 在 UI 中展示事件流、运行时间线、插件状态、审批请求和审计结果。

## 范围外或暂未定义

- 具体券商、交易所和交易 API 的接入范围。
- 每个行业包的完整数据源白名单和抓取频率。
- 交易策略的资金管理、仓位限制、止损止盈和合规审批细节。
- 生产环境的具体模型供应商账号、最终模型版本清单和推理成本预算阈值。
- 历史回测、相似事件检索和更完整的策略评估体系。

## 成功标准

- 系统能够稳定把外部事件路由到正确的行业包。
- 系统能够输出结构化的一级影响、二阶影响、风险提示和候选动作。
- 系统能够对高风险结论完成多信源验证。
- 系统能够按权限和风控策略进入正确的执行层级。
- 用户能够在 UI 中理解系统建议的依据，并能确认、拒绝或要求重新分析。

## 术语说明

| 术语 | 说明 |
| --- | --- |
| Source Plugin | 采集或接收外部信息的插件 |
| Industry Plugin | 承载行业分析和市场映射的插件 |
| AgentRuntime | 统一的 Agent 运行入口 |
| Registry | 插件与运行时状态索引 |
| Decision / Policy Gate | 决定通知、审批、dry-run 或执行的门禁 |
| HITL | Human-in-the-loop，人机协同确认机制 |
| WebSocket | 前端实时状态通知通道 |
