# 01. 产品概览

## 基本信息

**产品名称**：EventDrive-QuantAgent  
**版本**：v1.2.0  
**状态**：正式版整理稿  
**核心架构**：DeepAgents (LangGraph) + FastAPI + React

## 项目定义

EventDrive-QuantAgent 是一款面向专业交易者和研究员的事件驱动型量化智能系统。系统通过大语言模型作为推理核心，不仅监控表层新闻，还需要分析非结构化数据对特定行业的连锁影响。

产品核心能力是二阶推理：从事件本身出发，推导一级影响、二阶行业影响和可交易逻辑，并将分析结果转化为可执行或可确认的交易建议。

## 核心愿景

系统遵循 Plug-and-Play Industry Packages 理念。每个行业包都是一套独立的感知、逻辑与执行闭环，可被 Router Agent 按事件语义动态激活。

目标不是构建一个通用聊天机器人，而是构建一个能够围绕行业因果链完成事件监控、推理辩论、置信度评估、风控约束和交易建议输出的量化 Agent 系统。

## 目标用户

- 专业交易者：需要基于事件快速形成交易判断，并希望系统提供可解释的推理链路。
- 量化研究员：需要沉淀行业因果图谱，并验证事件信号的有效性。
- 策略开发者：需要以插件方式扩展行业包、数据源、金融工具映射和执行策略。

## 范围内能力

- 监控外部事件源，包括社交媒体、高权重官网、新闻源、论文平台和行业公开信息。
- 从非结构化文本中提取实体、事件、行业和潜在影响对象。
- 根据实体和语义将事件路由到一个或多个行业插件包。
- 使用行业逻辑图谱进行二阶推理。
- 通过多 Agent 内部辩论输出置信度评分。
- 根据置信度和市场执行权限进入自动执行、人工确认或仅通知模式。
- 在 UI 中实时展示事件流、推理摘要、置信度、建议动作和用户反馈入口。

## 范围外或暂未定义

- 未定义具体券商、交易所或交易 API 的接入范围。
- 未定义每个行业包的完整数据源白名单和抓取频率。
- 未定义交易策略的资金管理、仓位限制、止损止盈和合规审批细节。
- 未定义模型供应商、模型版本和推理成本预算。
- 未定义历史回测、相似事件检索等后续研究能力的范围和评价指标。

## 成功标准

- 系统能够把一个外部事件稳定路由到正确的行业包。
- 系统能够输出结构化的一级影响、二阶影响、交易逻辑和置信度。
- 系统能够对高风险结论完成至少两个独立信源的交叉验证。
- 系统能够按置信度和权限进入正确的执行层级。
- 用户能够在 UI 中理解系统为什么给出某个交易建议，并能确认、拒绝或反馈。

## 术语说明

| 术语 | 说明 |
| --- | --- |
| EventDrive | 事件驱动，以外部事件作为分析和交易触发源 |
| QuantAgent | 面向量化研究和交易决策的智能 Agent 系统 |
| 二阶推理 | 从事件直接影响进一步推导行业链、资产价格和策略动作的间接影响 |
| Industry Package | 行业插件包，包含感知器、逻辑图谱和策略适配器 |
| MCP | Model Context Protocol，用于行业插件向主 Agent 暴露工具和资源 |
| HITL | Human-in-the-loop，人机协同确认机制 |
