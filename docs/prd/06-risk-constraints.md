# 06. 风控、约束与异常处理

## 模块目标

本模块定义系统在推理、数据源、隐私、交易执行和噪音处理上的约束。由于系统输出可能影响真实交易，风控约束应优先于自动化效率。

## 幻觉校验

Agent 必须针对推理出的二阶影响搜索至少两个独立信源进行交叉验证。

### 要求

- 不允许基于单一社交媒体内容直接形成高置信度交易建议。
- 二阶影响需要有独立证据支持，例如新闻源、行业报告或官方数据。
- 验证失败时，系统应降低置信度或进入仅通知模式。
- 验证过程需要记录数据源、时间和摘要。

## 隐私保护

系统所有敏感配置仅存储于本地环境，不经过云端。

### 敏感信息范围

- API Keys。
- 交易账户信息。
- 交易策略参数。
- 用户自定义行业规则。
- 自动执行权限配置。

### 要求

- 敏感配置不得提交到代码仓库。
- 前端不得向非必要服务暴露敏感配置。
- 日志中不得输出完整密钥、账户凭证或私有策略。

## 噪音熔断

当单一账号短时间内发布大量信息时，系统需要自动识别为噪音并暂停该源的路由，直至人工解除。

### 要求

- 支持按来源维度统计事件频率。
- 支持按账号、域名、API 源或爬虫源触发熔断。
- 熔断后需要保留事件记录，但不继续进入高成本推理链路。
- 人工解除熔断时需要记录操作者和原因。

## 执行风险约束

### 自动执行前置条件

只有同时满足以下条件，系统才允许进入自动执行：

- 置信度 > 90%。
- 市场和账户具备 API 自动交易权限。
- 行业包和策略适配器标记该工具允许自动执行。
- 完成至少两个独立信源验证。
- 未触发噪音熔断、数据异常或交易限制。

### 禁止自动执行场景

- 无 API 权限市场。
- 数据源验证失败。
- 置信度低于阈值。
- 事件存在重大冲突信息。
- 账户、合规或风控开关禁用自动交易。

## 数据源异常

### 异常类型

- API 请求失败。
- 爬虫被反爬拦截。
- 数据源返回延迟或重复数据。
- 来源内容被删除或修改。
- 多信源结论冲突。

### 处理要求

- 记录失败原因和重试次数。
- 不因单一数据源失败直接中断整个系统。
- 对关键数据源失败进行 UI 提示。
- 多信源冲突时降低置信度，并进入人工确认或仅通知模式。

## Agent 推理异常

### 异常类型

- 无法提取关键实体。
- 无法匹配行业包。
- 行业包输出冲突。
- Internal Debate 输出缺少反方观点。
- 模型输出格式不符合结构化要求。

### 处理要求

- 尽量返回结构化错误，而不是自然语言模糊失败。
- 支持重新分析。
- 支持人工标注正确行业包。
- 记录失败输入和失败阶段，便于优化 Router Agent 和行业图谱。

## 审计要求

所有可能影响交易建议或执行的动作都需要可追溯。

至少记录：

- 事件 ID。
- 数据来源。
- 路由结果。
- 推理摘要。
- 验证来源。
- 置信度。
- 用户确认或拒绝记录。
- 执行动作和结果。
- 异常和熔断记录。
