# 03. 事件详情 / 决策页

## 页面定位

事件详情 / 决策页是操盘者围绕单条事件做判断的核心页面。它不是普通资讯详情页，也不是交易终端，而是把事件事实、行业影响分析、评分解释、最佳动作和审批入口串起来的决策页。

## 用户任务

- 确认事件事实和来源。
- 理解事件影响哪些行业、标的和风险方向。
- 查看系统给出的一个首版最佳动作。
- 查看支持观点、反方观点和不确定性。
- 进入审批、运行详情或事件审计时间线。

## 主对象和真源

主对象是 Event。

| 信息 | 真源 |
| --- | --- |
| 事件事实 | Event REST 快照 |
| 状态流转 | Event state transitions |
| 行业影响 | IndustryAnalysis / ScoredAnalysis |
| 最佳动作 | DecisionResult / ActionRequest 摘要 |
| 审批入口 | ApprovalRequest |
| 运行摘要 | AgentRun / ToolInvocation |
| 审计 | audit logs 和事件状态记录 |

## 页面结构

```text
页面头
  -> 左栏：事件事实和验证状态
  -> 右栏：行业影响分析和最佳动作
  -> 下方：支持/反方观点、运行摘要、审计入口
```

右栏是首屏重点，因为用户点入事件后最先需要判断“影响什么、系统建议什么”。

## 关键模块

### 事件事实

必须展示：

- 标题和摘要。
- 来源、来源类型、来源权威度。
- 发布时间和采集时间。
- 当前事件状态。
- 事件可信度和验证状态。
- 冲突信息摘要，如果存在。

### 行业影响分析

必须展示：

- 影响行业。
- 影响标的或市场对象。
- 影响方向。
- 影响持续时间。
- 行业影响强度。
- 风险点。
- 多行业共识和关键分歧，如果事件命中多个行业包。

### 最佳动作

首版只展示一个最佳动作。

必须展示：

- 建议标题。
- 目标标的或动作对象。
- 逻辑理由。
- 建议推荐度。
- 分析置信度。
- 风险等级和风险方向。
- 触发信息说明。
- 是否已产生 ApprovalRequest。

交互：

- `进入审批`：进入审批工作台或审批详情。
- `请求重分析`：如已支持，则需要填写或选择原因。
- 不提供本页直接批准或执行按钮。

### 支持 / 反方观点

必须展示结构化摘要：

- 支持观点。
- 反方观点。
- 证据质量。
- 数据缺口。

不展示完整 chain-of-thought 或原始长推理文本。

### 运行摘要

展示：

- 关联 AgentRun 数。
- 最近一次分析状态。
- 是否有关键工具失败。
- trace_id / request_id 入口。

点击进入 Runtime 详情页。

## 状态与失败路径

| 状态 | 页面行为 |
| --- | --- |
| captured / routed | 事实可见，分析区展示待分析 |
| analyzing | 事实可见，右栏展示分析中 |
| scored / decision_ready | 展示分析和最佳动作 |
| pending_approval | 审批入口高亮 |
| approved / rejected | 展示审批结果和审计入口 |
| failed | 展示失败阶段、错误摘要和重试入口 |
| 多信源冲突 | 降低可信度并展示冲突摘要 |
| 工具失败 | 在分析置信度和运行摘要中展示影响 |

## 风控边界

- 不在事件详情直接执行高风险动作。
- 不把“建议推荐度”写成“胜率”或“执行通过率”。
- 不展示 secret、私有策略、完整 prompt 或完整模型推理链。
- 用户拒绝后，同一建议不得继续自动进入 executor。

## 验收口径

必须成立：

- 用户能分清事件事实、分析结果和建议动作。
- 用户能知道最佳动作由哪条事件信息触发。
- 用户能看到反方观点和关键不确定性。
- 用户能进入审批和审计，不在本页绕过审批。

失败信号：

- 详情页只像新闻详情。
- 右侧分析和最佳动作不突出。
- 页面出现直接批准或执行动作。
- 用户无法解释建议为何产生。

## 非目标

- 不做多动作比较工作台。
- 不做交易图表终端。
- 不做本页直接 approve / execute。
- 不展示完整 chain-of-thought。
