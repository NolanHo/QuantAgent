# 01. 登录页

## 页面定位

登录页是操盘者进入 QuantAgent Web 工作台的系统入口。它的职责不是做复杂营销或品牌展示，而是让用户快速、安全地建立会话并进入事件分析主链路。

## 页面目标

- 让用户知道这是一个内部运行时管理台，而不是通用聊天或交易终端。
- 让用户快速完成身份验证。
- 明确当前环境状态，例如开发环境关闭鉴权、测试环境、正式环境。
- 登录成功后稳定跳转到 Dashboard `/`，再从 Dashboard 进入事件、审批或运行态。

## 入口与出口

### 入口

- 未登录用户访问任意受保护页面时跳转进入。
- 会话失效时自动跳转进入。
- 用户主动退出登录后进入。

### 出口

- 登录成功：跳转 `/`
- 登录失败：停留本页并展示错误
- 环境禁用鉴权：展示说明并允许直入 `/`

## 页面模块

| 模块 | 作用 | 主要内容 | 推荐前端模块 |
| --- | --- | --- | --- |
| 品牌说明区 | 解释系统用途 | 产品名、定位、风险提示 | `shared/ui/AuthHero` |
| 登录表单区 | 建立会话 | 用户名/邮箱、密码、提交按钮 | `features/auth/components/LoginForm` |
| 环境状态条 | 标明当前环境 | dev / staging / prod、鉴权状态 | `shared/ui/EnvironmentBanner` |
| 错误反馈区 | 提示失败原因 | 用户可读错误、request_id | `shared/ui/FormErrorNotice` |

## 功能明细

### 1. 品牌说明区

- 显示 `QuantAgent`
- 显示一句明确定位，例如：
  `事件驱动的量化分析与人工审批工作台`
- 显示一句风险说明，例如：
  `前端不直接放行高风险执行，所有高风险动作需经过审批链路。`

### 2. 登录表单区

首版建议字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 用户名或邮箱 | 是 | 系统登录身份 |
| 密码 | 是 | 本地会话或后端会话凭证 |

按钮：

- `登录`
- `进入开发模式`，仅在鉴权关闭环境展示

### 3. 环境状态条

至少展示：

- 当前环境名称
- 当前鉴权模式
- 是否允许开发直入

示例：

```text
当前环境：development
鉴权状态：已关闭
说明：当前环境允许直接进入管理台，仅供本地开发使用
```

### 4. 错误反馈区

常见错误：

- 用户名或密码错误
- 会话服务不可用
- 网络超时
- CSRF 或 cookie 配置异常

展示规则：

- 错误信息优先显示用户可理解文案
- 技术细节收纳在次要区域
- 如有 `request_id`，可在折叠区展示

## 页面示例

### 示例场景：本地开发

用户打开登录页，页面顶部出现：

```text
开发环境已关闭鉴权
你可以直接进入 Dashboard 进行本地验证
```

页面右侧仍保留表单，但主按钮变为：

- `进入 Dashboard`

### 示例场景：正式环境登录失败

用户输入错误密码后，表单下方显示：

```text
登录失败：用户名或密码错误
请检查凭证后重试
request_id: req_auth_3921
```

## 状态设计

| 状态 | 页面表现 |
| --- | --- |
| 初始 | 展示品牌区 + 表单 |
| 提交中 | 登录按钮 loading，禁用重复提交 |
| 失败 | 表单下方展示错误 |
| 成功 | 短暂提示后跳转 `/events` |
| 开发直入 | 显示环境横幅和直入按钮 |

## 推荐组件拆分

- `features/auth/components/LoginForm`
- `features/auth/components/EnvironmentBypassCard`
- `shared/ui/AuthHero`
- `shared/ui/FormErrorNotice`

## 非目标

- 不做注册页
- 不做找回密码流程
- 不做企业 SSO 多供应商切换器
- 不做品牌官网落地页
