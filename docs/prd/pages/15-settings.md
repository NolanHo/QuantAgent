# 15. Settings

## 页面定位

Settings 只承接会话、个人偏好和前端体验偏好。它不是插件配置中心、secret 管理中心、模型供应商配置中心、执行器权限中心或生产风控规则编辑器。

## 用户任务

- 查看当前身份、会话和环境。
- 退出登录。
- 管理个人通知提醒偏好。
- 管理前端刷新和实时连接提示偏好。
- 管理少量只影响展示的风险提醒偏好。

## 主对象和真源

| 信息 | 真源 |
| --- | --- |
| 当前身份 | `/me` 或 session snapshot |
| capability | 后端鉴权 / capability snapshot |
| UI 偏好 | 用户偏好配置，后续 contract 定义 |
| 实时连接状态 | runtime / realtime client 摘要 |

## 页面结构

```text
页面头
  -> 会话与身份
  -> 通知提醒偏好
  -> 实时连接和刷新偏好
  -> 展示偏好
```

## 允许设置

### 会话与身份

展示：

- actor。
- 环境。
- 登录状态。
- capability 摘要。
- 退出登录。

### 通知提醒偏好

只管理前端提醒体验：

- UI 内提醒开关。
- 声音提醒。
- 即将过期审批提醒。
- 高风险审批提醒。

外部通知渠道配置属于 notification 插件配置，不在 Settings 中管理。

### 实时连接和刷新偏好

允许：

- 是否显示断连提醒。
- 当前页面自动刷新间隔偏好。
- 是否在重连后自动刷新当前页面。

这些偏好不能改变 REST 作为状态真源的原则。

### 展示偏好

允许：

- 默认时间范围。
- 默认列表密度。
- 是否高亮 increase_risk。
- 是否默认折叠运行摘要。

## 不允许设置

Settings 不承接：

- 插件配置。
- secret reference 管理。
- API key 管理。
- LLM provider 管理。
- ProviderPolicy / 模型策略。
- 模型预算和 fallback 配置。
- executor 权限。
- 自动执行开关。
- 策略参数。
- 生产风控规则。
- Policy Gate 规则。

这些能力应分别进入 Plugin Detail、Model Providers / LLM Policies、后续 Secret 管理、Executor/Risk/Policy 专门治理入口，并必须有更强权限和审计。

## 状态与失败路径

| 状态 | 页面行为 |
| --- | --- |
| 未登录 | 引导登录 |
| 会话过期 | 展示重新登录入口 |
| 保存中 | 模块级 loading |
| 保存失败 | 模块级错误和 request_id |
| capability 不足 | 禁用相关偏好或隐藏入口 |

## 安全边界

- 不展示 token 原文。
- 不展示 secret。
- 不把高风险系统规则做成普通用户偏好。
- 前端偏好不能绕过后端权限校验。

## 验收口径

必须成立：

- 用户能管理个人使用体验。
- 用户不会在 Settings 里误以为可以控制插件、执行器或风控。
- 所有高风险系统能力有明确“不在本页”的归属说明。

失败信号：

- Settings 成为杂物间。
- 自动执行、secret、插件配置或模型策略被放进普通偏好。
- 用户可以通过 Settings 改变 Policy Gate 行为。

## 非目标

- 不做真实账户密钥管理中心。
- 不做插件配置中心。
- 不做模型供应商或模型策略管理中心。
- 不做策略引擎后台。
- 不做生产风控规则编辑器。
