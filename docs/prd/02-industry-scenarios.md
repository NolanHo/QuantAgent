# 02. 行业场景与插件包

## 模块目标

行业场景用于定义每个 Industry Plugin 的监控对象、典型触发事件、二阶分析路径和候选动作。每个行业包都应能通过 Source Binding、AgentDefinition、Skill 和 Tool 组合成独立的分析单元。

## 场景一：石油行业插件

**插件类型**：`industry`
**示例名称**：`Oil_Industry_Package`

### 监控目标

- X 高权重账号。
- OPEC+ 官网。
- 路透社能源板块。
- 红海和中东航运状态。

### 典型事件

政治人物发布消息，准备对伊朗石油出口实施更强制裁。

### 二阶分析路径

1. 一级影响：伊朗原油出口受阻，全球供应预期下降。
2. 二阶影响：地缘政治风险溢价上升，Brent / WTI 期限结构可能变化。
3. 候选动作：生成做多近月原油或相关能源资产的建议，由 Decision / Policy Gate 决定是否进入通知或审批。

### 需要沉淀的行业知识

- 石油供需、库存和期限结构变化。
- OPEC+ 增产、减产与会议结果。
- 地缘政治冲突对航运和保险成本的影响。
- Brent、WTI、成品油和能源股之间的联动。

### 主要风险

- 单一社交媒体内容噪音高。
- 供应减少的数量级需要独立验证。
- 期货和衍生品存在流动性、保证金和展期风险。

## 场景二：半导体 / 内存行业插件

**插件类型**：`industry`
**示例名称**：`Memory_Package`

### 监控目标

- arXiv AI 论文频道。
- Samsung 官方新闻稿。
- SK Hynix 官方新闻稿。
- 行业分析师公开言论。

### 典型事件

arXiv 发布论文，提出一种可显著降低推理显存占用的压缩算法。

### 二阶分析路径

1. 一级影响：模型对大容量显存的依赖下降。
2. 二阶影响：HBM、DDR5、GPU 采购节奏和存储库存预期变化。
3. 候选动作：生成对存储链或相关标的的看空/对冲建议，由 Decision / Policy Gate 决定执行层级。

### 需要沉淀的行业知识

- AI 推理成本、显存需求和硬件采购之间的关系。
- HBM、DDR5、GPU 和云厂商资本开支的关系。
- 技术论文从发布到产业落地的时间滞后。
- 存储厂商库存周期和财报指引变化。

### 主要风险

- 论文结果不等于可规模化商用。
- 技术利空可能被更强的需求增长抵消。
- 期权和对冲策略存在期限和流动性风险。

## 跨行业协同场景

当一个事件影响多个行业时，Router Agent 需要调度多个行业包并行分析。系统应支持输出多个独立分析结果，并在最终建议中区分直接影响、间接影响和冲突观点。

## 新增行业包模板

新增行业包时建议补齐以下信息：

| 字段 | 说明 |
| --- | --- |
| Package 名称 | 插件唯一标识 |
| 行业范围 | 覆盖的产业链和资产范围 |
| Source Binding | 依赖的数据源插件和配置模板 |
| AgentDefinition | 行业主 Agent 或子 Agent 定义 |
| Skill | 绑定的行业知识包 |
| Tool | 可用的受控工具 |
| Market Mapping | 标的、市场和候选动作映射 |
| 风险约束 | 验证、流动性、合规和噪音过滤 |
| 验收样例 | 至少一个输入事件和期望输出 |
