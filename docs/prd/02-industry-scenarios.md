# 02. 行业场景与插件包

## 模块目标

行业场景用于定义每个 Industry Package 的监控对象、典型触发事件、二阶推理路径和交易逻辑。每个行业包都应能够独立完成从事件感知到策略建议的闭环，并可被 Router Agent 动态调用。

## 场景一：石油行业插件

**插件名称**：`Oil_Industry_Package`

### 监控目标

- X 高权重账号，例如重要政治人物账号。
- OPEC+ 官网。
- 路透社能源板块。
- 红海航运状态。

### 典型事件

特朗普发布推文：准备对伊朗石油出口实施最高级别制裁。

### 二阶推理路径

1. 一级影响：伊朗原油出口受阻，全球供应可能直接减少约 100-200 万桶/日。
2. 二阶影响：地缘政治风险溢价上升，布伦特原油期限结构可能由 Contango 转为 Backwardation。
3. 交易逻辑：做多近月布伦特原油期货合约。

### 需要沉淀的行业知识

- 石油供需与库存变化对价格的影响。
- OPEC+ 增产、减产、会议结果与油价的历史关系。
- 地缘政治冲突对航运、保险成本和风险溢价的影响。
- Brent、WTI、成品油、能源股之间的联动关系。

### 主要风险

- 政治人物言论可能存在噪音、反转或策略性表达。
- 供应减少的数量级需要独立数据源验证，不能直接依赖单一文本。
- 期货交易存在展期、流动性和保证金风险。

## 场景二：半导体/内存行业插件

**插件名称**：`Memory_Package`

### 监控目标

- arXiv.org AI 论文频道。
- Samsung 官方新闻稿。
- SK Hynix 官方新闻稿。
- 行业分析师公开言论。

### 典型事件

arXiv 发布论文，提出一种新的压缩算法，可使大模型推理时的显存占用降低 70%。

### 二阶推理路径

1. 一级影响：模型对硬件显存大小的依赖度骤降。
2. 二阶影响：企业采购 HBM 和 DDR5 的紧迫性缓解，存储芯片厂商库存压力预期增加，未来季度财报指引可能下调。
3. 交易逻辑：做空美股存储相关标的，例如 Micron，或买入相应看跌期权。

### 需要沉淀的行业知识

- AI 推理成本、显存需求和硬件采购之间的关系。
- HBM、DDR5、GPU、服务器厂商与云厂商资本开支之间的关系。
- 技术论文从发布到产业落地的时间滞后。
- 存储厂商库存周期、价格周期和财报指引变化。

### 主要风险

- 论文结果不等于可规模化商用，需要验证代码、基准测试和行业采用可能性。
- 技术利空可能被更大的 AI 需求增长抵消。
- 期权策略涉及期限、波动率和流动性风险。

## 跨行业协同场景

当事件影响多个行业时，Router Agent 需要调度多个行业包协同分析。例如能源短缺可能影响半导体制造成本和供应链稳定性。

### 协同要求

- 支持一个事件激活多个 Industry Package。
- 支持不同 Package 输出独立影响链和置信度。
- 支持在最终建议中区分直接影响、间接影响和冲突观点。
- 支持将跨行业冲突提交给 Internal Debate 机制进行对比。

## 新增行业包模板

新增行业包时应补齐以下信息：

| 字段 | 说明 |
| --- | --- |
| Package 名称 | 插件唯一标识 |
| 行业范围 | 覆盖的产业链和资产范围 |
| 监控目标 | 数据源、账号、官网、API、频率 |
| 事件类型 | 触发该行业包的事件分类 |
| 逻辑图谱 | 因果链、正负影响、时间滞后 |
| 标的列表 | 股票、期货、期权、ETF、加密资产等 |
| 风险约束 | 数据验证、流动性、合规、噪音过滤 |
| 验收样例 | 至少一个输入事件和期望输出 |
