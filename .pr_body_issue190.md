## 关联

- Issue: #190
- OpenSpec change: `add-source-twelve-data-quote`

## 背景

`origin/main` 已经包含 Twelve Data quote 插件的首版实现，但还没有完全对齐维护者在 OpenSpec-only PR #192 中收紧后的最小契约，也保留了依赖当前仓库不存在调度模块的旧测试写法。本 PR 只补 issue #190 相对主线剩余的实现收口，不扩展到 runtime、scheduler 或其他行情能力。

## 改动

- Source Plugin
  - 为 Twelve Data quote 输出补齐批准后的最小 quote 契约：`source_plugin_id`、`source_type=market_quote`。
  - 保持 `external_id=provider:symbol:quote_timestamp`，继续在缺失时间戳时直接失败，不引入 fallback 标识。
- Tests
  - 重写 Twelve Data 测试为当前仓库真实可运行版本，只依赖现有 `registry` / `runtime` / `plugin-sdk`。
  - 删除对已不存在的 `quantagent.core.scheduling`、`quantagent.core.source_binding` 集成测试依赖。
  - 补充对 `source_type` / `source_plugin_id` 契约和缺失 `quote_timestamp` 失败路径的校验。
- Docs / OpenSpec
  - 同步 README，明确 latest quote 边界、secret 注入约束和 fail-fast 的 `quote_timestamp` 规则。
  - 将 `openspec/changes/add-source-twelve-data-quote/tasks.md` 按真实完成状态更新为已完成。

## 风险与未验证项

- 本 PR 没有改 runtime 的 entrypoint 加载策略；测试仍通过在测试环境注入插件根目录到 `sys.path` 来匹配当前仓库能力边界。
- 没有接入真实 Twelve Data 服务做联调，验证范围仅限 mock / fixture / 受控响应，这与 issue #190 的最小验证边界一致。

## Review Notes

- 这次没有继续修复或保留 scheduler 集成测试，不是遗漏，而是因为 `origin/main` 当前并不存在这些模块；继续保留只会让插件测试在主线上永久不可运行。
- `source_plugin_id` 和 `source_type` 被补到 `raw_payload` 与 `metadata`，是为了贴齐已批准的 OpenSpec 输出契约，而不是引入新的平台级 DTO。
