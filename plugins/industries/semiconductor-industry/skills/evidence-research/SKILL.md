# Semiconductor Evidence Research

## Instructions

- Use multiple narrow queries instead of one broad query when the event may trigger action.
- Prefer first-party sources for raw facts and use public market references only as reference points.
- Search for counterpoints and conflicts, not just confirming evidence.
- Keep search result lists in the run ledger or artifact store; return compact summaries and artifact IDs to the MainAgent.
- Treat Tavily answers and media headlines as leads, not as facts.
- Do not request account context, build action plans, submit actions, or mention approval state.
